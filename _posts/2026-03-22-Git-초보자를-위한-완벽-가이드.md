## Git이란 무엇이고, 왜 사용할까?

Git은 **분산 버전 관리 시스템(DVCS)**입니다. 쉽게 말하면, 파일의 변경 이력을 체계적으로 추적하고 관리해주는 도구입니다.

한 번 이런 경험을 떠올려보세요. `보고서_최종.docx`, `보고서_최종_수정.docx`, `보고서_진짜최종.docx`처럼 파일 이름을 바꿔가며 버전을 관리했던 적이 있지 않나요? Git은 바로 이 문제를 깔끔하게 해결해줍니다.

Git을 사용하면 다음과 같은 이점이 있습니다:

- **변경 이력 추적**: 누가, 언제, 무엇을 변경했는지 모든 기록이 남습니다.
- **협업 효율화**: 여러 개발자가 동시에 같은 프로젝트에서 작업할 수 있습니다.
- **안전한 실험**: 브랜치를 활용해 원본 코드에 영향 없이 새로운 기능을 시도할 수 있습니다.
- **롤백 가능**: 문제가 발생하면 언제든 이전 상태로 되돌릴 수 있습니다.

현재 전 세계 개발자의 90% 이상이 Git을 사용하고 있으며, GitHub, GitLab, Bitbucket 같은 플랫폼과 함께 사실상 업계 표준으로 자리잡았습니다.

## Git의 핵심 개념 이해하기

Git을 제대로 사용하려면 먼저 몇 가지 핵심 개념을 이해해야 합니다.

### 세 가지 영역: Working Directory, Staging Area, Repository

Git은 파일을 세 가지 영역으로 나누어 관리합니다.

1. **Working Directory (작업 디렉토리)**: 실제로 파일을 수정하는 공간입니다. 여러분이 코드를 작성하고 편집하는 바로 그 폴더입니다.
2. **Staging Area (스테이징 영역)**: 커밋에 포함시킬 변경사항을 임시로 모아두는 공간입니다. "이 변경사항들을 저장할 준비가 됐어"라고 표시하는 대기실 같은 곳이죠.
3. **Repository (저장소)**: 커밋된 변경 이력이 영구적으로 저장되는 공간입니다.

이 흐름을 정리하면 다음과 같습니다:

```
파일 수정 → git add (스테이징) → git commit (저장소에 기록)
```

### 브랜치(Branch)란?

브랜치는 독립적인 작업 공간을 만드는 기능입니다. 마치 평행 우주처럼, 메인 코드에 영향을 주지 않으면서 새로운 기능을 개발하거나 버그를 수정할 수 있습니다.

기본 브랜치는 보통 `main` 또는 `master`라고 부르며, 여기서 새 브랜치를 만들어 작업한 뒤 완료되면 다시 합치는(merge) 방식으로 진행합니다.

### 원격 저장소(Remote Repository)

로컬 컴퓨터에 있는 저장소 외에, GitHub 같은 서비스에 올려두는 저장소를 원격 저장소라고 합니다. 이를 통해 다른 개발자들과 코드를 공유하고 협업할 수 있습니다.

## 반드시 알아야 할 Git 필수 명령어

이제 실무에서 매일 사용하게 될 핵심 명령어들을 하나씩 살펴보겠습니다.

### 초기 설정

Git을 처음 설치했다면, 사용자 정보를 먼저 설정해야 합니다.

```bash
git config --global user.name "홍길동"
git config --global user.email "hong@example.com"
```

### 저장소 생성 및 복제

```bash
# 새 저장소 만들기
git init

# 기존 원격 저장소 복제하기
git clone https://github.com/username/repository.git
```

`git init`은 현재 폴더를 Git 저장소로 초기화하며, `git clone`은 원격에 있는 프로젝트를 내 컴퓨터로 통째로 가져옵니다.

### 일상적인 작업 흐름

```bash
# 현재 상태 확인
git status

# 변경된 파일을 스테이징 영역에 추가
git add 파일이름          # 특정 파일만
git add .                # 모든 변경사항

# 스테이징된 변경사항을 커밋
git commit -m "로그인 기능 구현 완료"

# 커밋 이력 확인
git log
git log --oneline
