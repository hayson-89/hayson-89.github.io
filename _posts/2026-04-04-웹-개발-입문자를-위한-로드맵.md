"웹 개발을 배우고 싶은데, 대체 어디서부터 시작해야 할까요?"

이 질문은 제가 가장 많이 받는 질문 중 하나입니다. 검색을 해보면 HTML, CSS, JavaScript, React, Node.js, Python, Django, Docker, AWS… 끝없이 쏟아지는 기술 용어에 압도당하기 쉽죠. 저도 처음 웹 개발을 시작했을 때 똑같은 막막함을 느꼈습니다.

오늘은 그런 분들을 위해, 2026년 기준으로 웹 개발 입문자가 어떤 순서로 무엇을 학습하면 좋을지 **현실적인 로드맵**을 정리해 보겠습니다. 이 글 하나로 최소 6개월에서 1년간의 학습 방향을 잡을 수 있을 거예요.

## 1단계: 웹의 기초 — HTML, CSS, JavaScript

어떤 프레임워크를 배우든, 어떤 분야로 진출하든 이 세 가지는 반드시 거쳐야 하는 기본 중의 기본입니다.

### HTML (1~2주)

HTML은 웹 페이지의 **뼈대**입니다. 제목, 문단, 이미지, 링크, 폼 등 웹 페이지를 구성하는 요소들을 마크업하는 언어죠. 프로그래밍 언어라기보다는 구조를 정의하는 언어이기 때문에 진입 장벽이 매우 낮습니다.

- 시맨틱 태그(`<header>`, `<nav>`, `<main>`, `<article>` 등)의 의미를 이해하세요.
- 폼(form) 요소를 다루는 방법을 익히세요.
- 추천 자료: MDN Web Docs, 생활코딩 HTML 편

### CSS (2~3주)

CSS는 웹 페이지의 **옷**입니다. 색상, 폰트, 레이아웃, 애니메이션 등 시각적인 모든 것을 담당합니다.

- **Flexbox**와 **Grid** 레이아웃은 반드시 마스터하세요. 이 두 가지만 잘 알아도 대부분의 레이아웃을 구현할 수 있습니다.
- 반응형 웹 디자인(미디어 쿼리)도 초반에 익혀두면 좋습니다.
- 추천 자료: CSS-Tricks, Flexbox Froggy(게임으로 배우는 Flexbox)

### JavaScript (4~6주)

JavaScript는 웹 페이지에 **생명**을 불어넣는 언어입니다. 버튼 클릭, 데이터 처리, 서버 통신 등 동적인 기능을 모두 JavaScript로 구현합니다.

- 변수, 조건문, 반복문, 함수 등 기본 문법부터 시작하세요.
- DOM 조작(문서 객체 모델)을 통해 HTML 요소를 동적으로 변경하는 방법을 배우세요.
- `fetch` API를 활용한 비동기 통신(API 호출)까지 다룰 수 있으면 1단계는 완료입니다.
- 추천 자료: JavaScript.info, 드림코딩 유튜브 채널

> **💡 팁:** 이 단계에서 간단한 프로젝트를 하나 만들어 보세요. 할 일 목록(To-Do List), 날씨 앱, 간단한 계산기 등 작지만 완성된 무언가를 만들어 보는 경험이 정말 중요합니다.

## 2단계: 프론트엔드 또는 백엔드, 방향을 선택하자

기초를 어느 정도 다졌다면, 이제 방향을 정해야 합니다. 웹 개발은 크게 **프론트엔드**와 **백엔드**로 나뉩니다.

### 프론트엔드 (사용자가 보는 화면)

사용자와 직접 상호작용하는 인터페이스를 개발합니다. 디자인 감각이 있거나, 눈에 보이는 결과물에 동기부여를 받는 분에게 추천합니다.

- **React** — 2026년 현재에도 가장 높은 점유율을 가진 프론트엔드 라이브러리입니다. 취업 시장에서의 수요도 가장 많습니다.
- **Next.js** — React 기반의 풀스택 프레임워크로, 서버 사이드 렌더링(SSR)과 정적 사이트 생성(SSG)을 지원합니다. React를 어느 정도 익혔다면 자연스럽게 넘어가세요.
- **TypeScript** — JavaScript에 타입 시스템을 추가한 언어입니다. 현업에서는 거의 필수로 사용되므로 일찍 익혀두면 좋습니다.
