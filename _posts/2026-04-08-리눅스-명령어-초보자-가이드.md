## 파일과 디렉토리 다루기: 가장 먼저 배워야 할 명령어들

리눅스에서 가장 자주 사용하는 작업은 파일과 폴더를 탐색하고 관리하는 것입니다. 윈도우의 파일 탐색기에서 클릭으로 하던 일을, 터미널에서는 짧은 명령어로 처리합니다.

### `pwd` — 현재 위치 확인

```bash
$ pwd
/home/username
```

"Print Working Directory"의 약자입니다. 내가 지금 어디에 있는지 길을 잃었을 때 가장 먼저 입력하세요.

### `ls` — 파일 목록 보기

```bash
$ ls          # 현재 디렉토리의 파일 목록
$ ls -la      # 숨김 파일 포함, 상세 정보 표시
$ ls -lh      # 파일 크기를 읽기 쉬운 단위(KB, MB)로 표시
```

`-la` 옵션은 실무에서 가장 많이 쓰는 조합입니다. 파일 권한, 소유자, 수정 날짜까지 한눈에 확인할 수 있거든요.

### `cd` — 디렉토리 이동

```bash
$ cd /var/log        # 절대 경로로 이동
$ cd ..              # 상위 디렉토리로 이동
$ cd ~               # 홈 디렉토리로 바로 이동
$ cd -               # 직전 디렉토리로 되돌아가기
```

특히 `cd -`는 두 디렉토리를 왔다 갔다 할 때 정말 편리합니다. 의외로 모르는 분들이 많은 꿀팁이에요.

### `mkdir`, `cp`, `mv`, `rm` — 생성, 복사, 이동, 삭제

```bash
$ mkdir my_project             # 디렉토리 생성
$ mkdir -p parent/child/grand  # 중간 경로까지 한 번에 생성
$ cp file.txt backup.txt       # 파일 복사
$ cp -r folder1 folder2        # 디렉토리 전체 복사
$ mv old.txt new.txt           # 이름 변경 또는 이동
$ rm file.txt                  # 파일 삭제
$ rm -rf directory             # 디렉토리 강제 삭제 (⚠️ 주의!)
```

> ⚠️ **경고**: `rm -rf`는 확인 없이 모든 것을 삭제합니다. 특히 `sudo rm -rf /` 같은 명령은 시스템 전체를 날릴 수 있으니, 삭제 전에 항상 경로를 두 번 확인하세요.

## 파일 내용 확인과 검색: 로그 분석의 기본기

서버 운영이든 개발이든, 파일 내용을 빠르게 확인하고 원하는 정보를 찾아내는 능력은 핵심입니다.

### `cat`, `head`, `tail` — 파일 내용 읽기

```bash
$ cat config.yaml          # 파일 전체 내용 출력
$ head -20 access.log      # 처음 20줄만 출력
$ tail -50 error.log       # 마지막 50줄만 출력
$ tail -f application.log  # 실시간으로 추가되는 로그 모니터링
```

`tail -f`는 서버 로그를 실시간으로 모니터링할 때 필수적인 명령어입니다. 장애 대응 시 가장 먼저 실행하게 되는 명령어이기도 하죠.

$ grep "ERROR" application.log                  # ERROR가 포함된 줄 찾기

$ grep -r "TODO" ./src/                         # 디렉토리 내 재귀 검색
$ grep -n "database" config.yaml                # 줄 번호와 함께 출력
