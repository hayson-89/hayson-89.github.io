블로그를 운영하다 보면 문득 이런 궁금증이 생깁니다. "내 글을 도대체 누가, 어디서, 어떻게 읽고 있는 걸까?" 열심히 글을 쓰는 것도 중요하지만, 데이터를 기반으로 방향을 잡는 것이야말로 블로그를 꾸준히 성장시키는 핵심 비결입니다. 오늘은 전 세계에서 가장 널리 쓰이는 무료 웹 분석 도구인 **구글 애널리틱스(Google Analytics)**를 활용해 블로그 트래픽을 분석하는 방법을 단계별로 알아보겠습니다.

## 구글 애널리틱스, 왜 써야 할까?

블로그 플랫폼 자체에서 제공하는 기본 통계만으로는 한계가 있습니다. 티스토리나 워드프레스 대시보드에서 일별 방문자 수 정도는 확인할 수 있지만, **방문자가 어디에서 왔는지**, **어떤 글에서 이탈했는지**, **모바일과 데스크톱 비율이 어떤지** 같은 깊이 있는 인사이트를 얻기는 어렵습니다.

구글 애널리틱스를 사용하면 다음과 같은 것들을 무료로 파악할 수 있습니다:

- **실시간 접속자 수**와 그들이 보고 있는 페이지
- 방문자의 **지역, 언어, 기기 정보**
- **유입 경로** (검색엔진, SNS, 직접 방문, 다른 사이트 링크 등)
- 각 페이지별 **체류 시간**과 **이탈률**
- 시간대별, 요일별 **트래픽 패턴**

이 데이터들은 단순한 숫자가 아닙니다. "어떤 주제의 글을 더 써야 하는지", "어떤 채널에 홍보를 집중해야 하는지"를 결정하는 **의사결정의 근거**가 됩니다.

## 설치부터 기본 설정까지 — 10분이면 충분합니다

구글 애널리틱스를 처음 시작하는 분들을 위해 설정 과정을 간단히 정리해 보겠습니다. 현재 구글이 제공하는 최신 버전은 **GA4(Google Analytics 4)**입니다.

**1단계: 계정 생성**
[analytics.google.com](https://analytics.google.com)에 접속해서 구글 계정으로 로그인한 뒤 "측정 시작" 버튼을 클릭합니다. 계정 이름, 속성 이름(블로그 이름), 시간대와 통화를 설정하면 기본 계정이 만들어집니다.

**2단계: 데이터 스트림 설정**
플랫폼으로 "웹"을 선택하고 블로그 URL을 입력합니다. 그러면 **측정 ID** (G-XXXXXXXXXX 형태)가 발급됩니다.

**3단계: 추적 코드 삽입**
발급받은 측정 ID를 블로그에 삽입해야 합니다.

- **워드프레스**: Site Kit by Google 플러그인을 설치하면 클릭 몇 번으로 연동됩니다. 또는 테마의 `header.php`에 직접 추적 코드를 붙여넣을 수도 있습니다.
- **티스토리**: 관리자 페이지 → 꾸미기 → 스킨 편집 → HTML에서 `<head>` 태그 안에 추적 코드를 삽입합니다.
- **Jekyll / Hugo 등 정적 사이트**: 레이아웃 템플릿의 head 영역에 코드를 추가합니다.

설치 후 **실시간 보고서**에서 자신의 접속이 감지되면 정상적으로 작동하는 것입니다. 여기까지 하는 데 보통 10분이면 충분합니다.

**꿀팁**: 본인의 방문이 데이터에 섞이지 않도록 **내부 트래픽 필터**를 설정하는 것을 추천합니다. GA4 관리 → 데이터 스트림 → 태그 설정에서 내부 트래픽 정의를 통해 자신의 IP 주소를 제외할 수 있습니다.
