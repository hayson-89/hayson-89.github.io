VS Code는 이미 그 자체로 훌륭한 에디터입니다. 하지만 확장 프로그램을 제대로 활용하면, 마치 일반 자동차에 터보 엔진을 얹은 것처럼 생산성이 급격히 달라집니다. 저도 개발을 시작한 초기에는 "확장 프로그램 없이도 충분하지 않나?"라고 생각했지만, 하나둘 설치하면서 "왜 진작 안 썼지?"라는 후회를 수없이 했습니다.

오늘은 수많은 확장 프로그램 중에서 **실무에서 실제로 생산성 향상에 도움이 되는 것들만** 엄선해서 소개해 드리겠습니다. 카테고리별로 나누어 정리했으니, 본인의 작업 스타일에 맞는 것부터 하나씩 설치해 보세요.

## 코드 품질과 자동 완성을 책임지는 확장 프로그램

생산성의 첫 번째 핵심은 **코드를 빠르고 정확하게 작성하는 것**입니다. 다음 확장 프로그램들은 코드 작성 속도와 품질을 동시에 끌어올려 줍니다.

### 1. GitHub Copilot
2026년 현재 가장 강력한 AI 코드 어시스턴트입니다. 주석을 작성하면 함수 전체를 자동 생성해 주고, 반복적인 보일러플레이트 코드를 순식간에 완성해 줍니다. 월 구독 비용이 있지만, 절약되는 시간을 생각하면 투자 대비 효과가 압도적입니다. 특히 테스트 코드 작성이나 데이터 변환 로직처럼 패턴이 반복되는 작업에서 진가를 발휘합니다.

### 2. ESLint & Prettier
이 두 가지는 이제 선택이 아니라 필수입니다. **ESLint**는 코드의 잠재적 오류와 안티패턴을 실시간으로 잡아주고, **Prettier**는 코드 포맷팅을 자동으로 통일해 줍니다. 팀 프로젝트에서 "탭이냐 스페이스냐" 같은 불필요한 논쟁을 원천 차단할 수 있죠. 저장 시 자동 포맷팅을 설정해 두면 코드 스타일을 신경 쓰는 데 드는 인지 부하를 완전히 없앨 수 있습니다.

```json
// settings.json에 추가
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode"
}
```

### 3. IntelliCode
마이크로소프트가 만든 AI 기반 자동 완성 도구입니다. 일반적인 자동 완성과 달리, 수천 개의 오픈소스 프로젝트에서 학습한 패턴을 기반으로 **가장 사용할 가능성이 높은 API를 상단에 추천**해 줍니다. Copilot과 함께 쓰면 시너지가 뛰어납니다.

### 4. Error Lens
에러나 경고 메시지를 해당 코드 라인 옆에 인라인으로 바로 표시해 줍니다. 기존에는 문제 탭을 열거나 빨간 밑줄 위에 마우스를 올려야 했는데, 이 확장 프로그램을 쓰면 **코드를 읽는 흐름을 끊지 않고도 즉시 문제를 파악**할 수 있습니다. 한번 써보면 없이는 못 사는 확장 프로그램입니다.

## 작업 효율을 극대화하는 유틸리티 확장 프로그램

코드 작성 외에도, 개발 워크플로우 전반의 효율을 높여주는 도구들이 있습니다.

### 5. GitLens
VS Code에 내장된 Git 기능도 괜찮지만, GitLens는 차원이 다릅니다. 각 코드 라인별로 **누가 언제 왜 수정했는지**(Git Blame)를 인라인으로 보여주고, 파일의 변경 이력을 시각적으로 탐색할 수 있게 해줍니다. "이 코드 왜 이렇게 작성했지?"라는 의문이 들 때, 커밋 메시지를 바로 확인할 수 있어 디버깅 시간이 크게 줄어듭니다.

### 6. Thunder Client
API 테스트를 위해 Postman을 별도로 열 필요 없이, VS Code 안에서 바로 REST API 요청을 보내고 응답을 확인할 수 있습니다. 가볍고 빠르며, 컬렉션 관리와 환경 변수 설정도 지원합니다. **에디터와 API 테스트 도구 사이를 왔다 갔다 하는 컨텍스트 스위칭 비용**을 확실히 줄여줍니다.
