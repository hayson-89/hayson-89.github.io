처음 리눅스 터미널을 열었을 때의 그 막막한 느낌, 혹시 기억하시나요? 까만 화면에 깜빡이는 커서만 보이고, 무엇을 입력해야 할지 전혀 감이 오지 않던 그 순간 말입니다. 저도 그랬습니다. 하지만 걱정하지 마세요. 리눅스 명령어는 생각보다 체계적이고, 핵심 명령어 몇 가지만 익히면 터미널이 오히려 마우스보다 편해지는 날이 반드시 옵니다. 오늘은 리눅스를 처음 접하는 분들을 위해, 반드시 알아야 할 기본 명령어들을 실전 예제와 함께 정리해 보겠습니다.

## 파일과 디렉토리 다루기: 가장 먼저 익혀야 할 명령어들

리눅스에서 가장 자주 하는 작업은 파일과 폴더를 탐색하고 관리하는 것입니다. 윈도우에서 탐색기로 하던 일을 터미널에서 텍스트로 수행한다고 생각하면 됩니다.

**`pwd` — 현재 위치 확인**

터미널을 열면 가장 먼저 "나는 지금 어디에 있지?"라는 질문이 떠오릅니다. 이때 `pwd`(Print Working Directory)를 입력하면 현재 작업 디렉토리의 전체 경로를 보여줍니다.

```bash
$ pwd
/home/username
```

**`ls` — 파일 목록 보기**

현재 디렉토리에 어떤 파일과 폴더가 있는지 확인하는 명령어입니다. 가장 많이 쓰는 명령어 중 하나이며, 다양한 옵션을 붙여서 활용할 수 있습니다.

```bash
$ ls          # 기본 목록 보기
$ ls -l       # 상세 정보(권한, 크기, 수정일 등) 포함
$ ls -la      # 숨겨진 파일(.으로 시작하는 파일)까지 모두 표시
$ ls -lh      # 파일 크기를 사람이 읽기 쉬운 단위(KB, MB)로 표시
```

**`cd` — 디렉토리 이동**

폴더 사이를 이동할 때 사용합니다. 윈도우 탐색기에서 폴더를 더블클릭하는 것과 같은 동작입니다.

```bash
$ cd Documents        # Documents 폴더로 이동
$ cd /var/log         # 절대 경로로 이동
$ cd ..               # 상위 디렉토리로 이동
$ cd ~                # 홈 디렉토리로 바로 이동
$ cd -                # 직전에 있던 디렉토리로 돌아가기
```

**`mkdir` / `rmdir` — 디렉토리 생성과 삭제**

```bash
$ mkdir my_project           # my_project 폴더 생성
$ mkdir -p parent/child/sub  # 중간 경로까지 한 번에 생성
$ rmdir my_project           # 빈 디렉토리 삭제
```

**`cp`, `mv`, `rm` — 복사, 이동, 삭제**

```bash
$ cp file.txt backup.txt         # 파일 복사
$ cp -r my_folder backup_folder  # 폴더 전체 복사 (-r: recursive)
$ mv file.txt Documents/         # 파일 이동
$ mv old_name.txt new_name.txt   # 파일 이름 변경
$ rm file.txt                    # 파일 삭제
$ rm -rf my_folder               # 폴더와 내용 전체 강제 삭제 (주의!)
```

> ⚠️ **주의:** `rm -rf`는 확인 없이 모든 것을 삭제합니다. 특히 `rm -rf /`와 같은 명령은 시스템 전체를 날릴 수 있으니 절대 실행하지 마세요!

## 파일 내용 확인과 검색: 터미널에서 텍스트 다루기

GUI 환경에서는 파일을 더블클릭해서 열어보지만, 터미널에서는 명령어로 파일 내용을 빠르게 확인할 수 있습니다. 오히려 대용량 파일을 다룰 때는 터미널이 훨씬 효율적입니다.

**`cat` — 파일 전체 내용 출력**

```bash
$ cat readme.txt               # 파일 전체 내용 출력
$ cat file1.txt file2.txt      # 여러 파일을 연결해서 출력
```

짧은 파일을 빠르게 확인할 때 유용하지만, 파일이 길면 화면이 순식간에 지나갑니다.

**`head` / `tail` — 파일의 처음과 끝 확인**

```bash
$ head -n 20 logfile.txt    # 처음 20줄만 출력
$ tail -n 50 logfile.txt    # 마지막 50줄만 출력
$ tail -f /var/log/syslog   # 실시간으로 로그 파일 모니터링
```

`tail -f`는 서버 로그를 실시간으로 확인할 때 정말 많이 사용하는 명령어입니다. 개발자라면 반드시 기억해 두세요
