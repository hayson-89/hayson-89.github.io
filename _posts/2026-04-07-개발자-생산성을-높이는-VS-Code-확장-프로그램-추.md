VS Code는 이미 그 자체로 강력한 에디터입니다. 하지만 확장 프로그램을 제대로 갖추면, 마치 일반 자전거에서 전동 킥보드로 갈아탄 것 같은 속도감을 느낄 수 있죠. 저도 수년간 수많은 확장 프로그램을 설치하고 지우기를 반복하면서, 결국 "이것만은 절대 지우지 않는다"는 리스트가 만들어졌습니다.

오늘은 그 리스트를 여러분과 나눠보려 합니다. 단순히 인기 많은 것이 아니라, **실제로 매일 쓰면서 체감 생산성이 올라간** 확장 프로그램만 엄선했습니다.

## 코드 작성 속도를 끌어올리는 확장 프로그램

개발의 가장 기본은 코드를 작성하는 행위 그 자체입니다. 이 과정을 빠르고 정확하게 만들어 주는 확장 프로그램부터 살펴보겠습니다.

**1. GitHub Copilot**

2026년 현재, AI 코드 어시스턴트를 안 쓰는 개발자를 찾기가 더 어려워졌습니다. GitHub Copilot은 여전히 이 분야의 대표 주자입니다. 단순 자동 완성을 넘어서 함수 전체를 제안해 주고, 주석만 써도 코드를 생성해 줍니다. 특히 반복적인 보일러플레이트 코드를 작성할 때 진가를 발휘합니다. 월 구독 비용이 들지만, 절약되는 시간을 생각하면 투자 대비 효과가 확실합니다.

**2. IntelliCode**

Microsoft가 만든 AI 기반 자동 완성 도구입니다. Copilot과 다른 점은 여러분의 코드 패턴을 학습해서 가장 가능성 높은 API를 상위에 추천해 준다는 것입니다. 자동 완성 목록에서 원하는 항목을 찾느라 스크롤하던 시간이 눈에 띄게 줄어듭니다. 무료라는 점도 큰 장점이죠.

**3. Tabnine**

Copilot의 대안을 찾는 분들에게 추천합니다. 로컬 모델을 지원하기 때문에 회사 보안 정책상 클라우드 기반 AI를 사용할 수 없는 환경에서도 쓸 수 있습니다. 코드가 외부 서버로 전송되지 않는다는 점에서 프라이버시에 민감한 팀에 적합합니다.

## 코드 품질과 가독성을 지켜주는 확장 프로그램

빠르게 작성하는 것만큼 중요한 것이 깔끔하게 작성하는 것입니다. 아래 확장 프로그램들은 코드 리뷰 시간을 줄이고, 팀 전체의 코드 일관성을 유지하는 데 큰 도움이 됩니다.

**4. ESLint**

JavaScript/TypeScript 개발자라면 필수 중의 필수입니다. 코드를 저장하는 순간 문법 오류와 스타일 위반을 잡아줍니다. `.eslintrc` 설정 파일을 팀 차원에서 공유하면, 모든 팀원이 동일한 규칙으로 코드를 작성하게 되어 코드 리뷰에서 스타일 관련 논쟁이 사라집니다.

**5. Prettier - Code Formatter**

코드 포맷팅은 사람이 할 일이 아닙니다. Prettier를 설치하고 "저장 시 자동 포맷팅" 옵션만 켜두면, 들여쓰기·따옴표·세미콜론 같은 사소한 고민에서 완전히 해방됩니다. ESLint와 함께 쓰면 금상첨화입니다.

**6. Error Lens**

기본 VS Code에서는 에러가 있는 줄에 밑줄만 살짝 그어져서 놓치기 쉽습니다. Error Lens는 에러 메시지를 해당 줄 바로 옆에 인라인으로 표시해 줍니다. 빨간 글씨가 코드 바로 옆에 보이니, 문제를 즉시 인지하고 수정할 수 있습니다. 한번 써보면 없이는 못 살게 되는 확장 프로그램입니다.

**7. SonarLint**

보안 취약점과 코드 스멜(Code Smell)을 실시간으로 감지해 줍니다. 단순 문법 오류를 넘어서 "이 코드는 왜 문제가 되는지"까지 설명해 주기 때문에 학습 효과도 있습니다. 시니어 개발자가 옆에서 코드 리뷰해 주는 느낌이랄까요.
